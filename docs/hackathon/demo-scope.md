# Hackathon Demo Scope

The full repository supports the complete product workflow, but the hackathon demo should show one polished end-to-end path.

## Recommended Demo Path

1. Start with either an inbound pitch upload or a preloaded GitHub/Product Hunt signal.
2. Show the signal becoming an activated founder/company record.
3. Collect evidence from a small set of sources.
4. Run the intelligence layer:
   - entity resolution
   - thesis matching
   - founder score
   - market score
   - idea-vs-market stress test
   - evidence/trust verification
   - memo generation
   - adversarial review
5. Show the investor workspace:
   - ranked dashboard
   - company profile
   - founder profile
   - evidence explorer
   - scorecards
   - recommendation
   - cited investment memo
   - adversarial view
6. Record a pass/invest decision and write a memory update.

## MVP Bias

Prioritize evidence traceability, explainable scores, and a professional memo over broad connector coverage. One real connector plus high-quality fixtures is better than many shallow integrations.
